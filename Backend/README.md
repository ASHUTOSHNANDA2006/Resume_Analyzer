# ⚙️ InterviewIntel - Backend REST API

This is the robust Express.js API backend powering **InterviewIntel**, the AI Resume Analyzer. It processes multi-part PDF resumes, manages user authentication using JWT cookies with token blacklisting, integrates with the new `@google/genai` Gemini SDK, and orchestrates Puppeteer headless browser pipelines to compile downloadable, ATS-friendly A4 resumes.

---

## 🚀 Technology Stack & Services

- **Runtime Environment:** Node.js (v18+)
- **Server Framework:** Express.js (supporting middleware-driven route execution)
- **Database Layer:** MongoDB Atlas with Mongoose ORM
- **AI Engine:** Google Gemini API via the official `@google/genai` SDK (running `gemini-2.5-flash`)
- **PDF Compiler:** Puppeteer (headless Chrome automation)
- **Parser Engine:** `pdf-parse` (extracts plain text from raw PDF buffers)
- **File Upload Middleware:** Multer (supporting memory-stored `multipart/form-data`)
- **Security:** `bcryptjs` (password hashing) and `jsonwebtoken` (session handling)

## 🔐 Database Models

### 1. User Schema (`users`)
- `username`: String (Unique, Required)
- `email`: String (Unique, Required, Format validated)
- `password`: String (Required, Hashed with Bcrypt salts)

### 2. Token Blacklist Schema (`blacklistTokens`)
- `token`: String (Required, Indexed)
- `createdAt`: Date (Auto-expires using Mongoose TTL index to purge expired session keys)

### 3. Interview Report Schema (`interviewreports`)
- `userId`: ObjectId (Relation mapping to user creator)
- `title`: String (Job Title)
- `matchScore`: Number (0-100 rating)
- `atsScore`: Number (0-100 rating)
- `technicalQuestions`: Array of `{ question, intention, answer }`
- `behavioralQuestions`: Array of `{ question, intention, answer }`
- `skillGaps`: Array of `{ skill, severity: "Low" | "Medium" | "High" }`
- `preparationPlan`: Array of `{ day, focus, tasks: String[] }`
- `createdAt`: Date

## 🔌 API Endpoints

### 1. Authentication (`/api/auth`)

#### `POST /api/auth/register`
Creates a new account and sets a JWT cookie header automatically.
- **Request Body:**
  ```json
  {
    "username": "johndoe",
    "email": "johndoe@example.com",
    "password": "mysecurepassword"
  }
  ```
- **Response (201 Created):**
  ```json
  {
    "message": "User registered successfully",
    "user": { "id": "user_id", "username": "johndoe", "email": "johndoe@example.com" }
  }
  ```

#### `POST /api/auth/login`
Validates credentials and responds with a secure HTTP-Only JWT session cookie.
- **Request Body:**
  ```json
  {
    "email": "johndoe@example.com",
    "password": "mysecurepassword"
  }
  ```

#### `GET /api/auth/logout`
Blacklists the active JWT token inside the database `blacklistTokens` collection and wipes the client's cookie.
- **Response (200 OK):**
  ```json
  { "message": "User logged out successfully" }
  ```

#### `GET /api/auth/get-me`
Fetches authenticated user information. Requires active session cookie.

### 2. Interview & Resume Suite (`/api/interview`)

#### `POST /api/interview/`
Generates a complete structured interview preparation roadmap by processing user profiles against job descriptions.
- **Request Headers:** `Content-Type: multipart/form-data`
- **Form Data:**
  - `resume`: File (`.pdf` format)
  - `jobDescription`: String
  - `selfDescription`: String
- **AI Processing Pipeline:**
  1. Multer uploads file buffer.
  2. `pdf-parse` extracts raw text from PDF.
  3. Context parameters are sent to the `@google/genai` structured JSON parser.
  4. Returns type-safe, validated schemas (Scores, Skill Gaps, Roadmaps).
- **Response (201 Created):**
  ```json
  {
    "message": "Report generated successfully",
    "report": { ...interviewReportSchemaDetails }
  }
  ```

#### `GET /api/interview/`
Fetches a list of all historical reports generated by the active user.

#### `GET /api/interview/report/:interviewId`
Retrieves a detailed report by its unique MongoDB identifier.

#### `POST /api/interview/resume/pdf/:interviewReportId`
Compiles a customized, ATS-optimized, high-impact resume PDF tailored directly to the report's target job description.
- **Underlying Logic:**
  1. Fetches stored report context parameters.
  2. Uses `gemini-2.5-flash` to construct a customized, professional HTML resume incorporating inline CSS styling.
  3. Launches a headless **Puppeteer** instance to compile the generated HTML template into a standard A4 PDF document.
  4. Responds with binary PDF content, directly triggerable as a local download in the user's browser.
